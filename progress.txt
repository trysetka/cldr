### 2026-03-25 — Phase 2 Complete

**Beads task:** cldr-z1l: Phase 2: Backend — Locale Validation
**Completed:** RuntimeStore fallback for locale validation, known_locale_names, known_locale_name?
**Phase reference:** PLAN.md §Phase 2

**Key decisions:**
- Added `do_validate_locale/1` fallback that checks RuntimeStore before `Cldr.Locale.new/2` full parse path
- The fallback handles both string and atom inputs via separate guard clauses (`is_atom`/`is_binary`), delegates to shared `do_validate_locale_for_string/2` helper
- String locale names are converted to atoms for RuntimeStore lookup (`String.to_atom/1`) — safe because loaded locale atoms already exist
- LanguageTag is constructed via `Cldr.Config.language_tag/1` + `Cldr.Locale.put_gettext_locale_name/2` + `Map.put(:backend, ...)` — mirrors compiled path
- `@cldr_config` module attribute stores the compile-time config struct for use in `defp` (bind_quoted `config` variable has context issues in function defs)
- `known_locale_names/0` extended to merge `@known_locale_names` with `RuntimeStore.known_loaded_locales(__MODULE__)`
- `known_locale_name?/1` automatically works via its delegation to the extended `known_locale_names/0`
- Used `NoFallback.Cldr` (locales: ["en", "es-US"]) for integration tests to exercise the fallback path
